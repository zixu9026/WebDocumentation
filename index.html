Landing Page

For the landing page, my idea was to explore some background information about the charity and current news about the reef. In the initial ideation of wireframe and mockup, the navigation bar had some user interactions such as user profile, liked articles and search bar. However, I decided to make the navigation bar simple in the web prototype. The buttons were moved to the right. The donate button was different style. It can be an emphasise to the users this button is important.

Under each news article, there is a 'Learn More' link that users can read the full article. There is also a heart icon followed that users can click to liked the article. The more liked articles will appear in the home page of the charity website.

The footer is the section that users can get social media links and contact details when they scroll to the bottom of the website. I provided the social media links and contact number of the charity. On the right is a login section. It's an invitation to users to join the reef-saving community. It requires user inputs that users can provide their personal details and get informations from the website. For the footer, I think there can be some inprovements that more information provide to users will be better, such as: links to other pages in the website.

For the mobile version, I focused more on making a responsive navigation bar.

Project Page

For the project page, the original idea was put different projects in a column. However, in the coding process. I find out that when users first enter to this page, they will only see one project at top. This may influence their thoughts and don't want to scroll down. So I decided to put the projects in two columns that users can see different projects at the first time.

The progress bar was used to show the donation progress visually. I have searched how to make a css progress bar but it didn't work for me. So I just put an image instead to demonstrate the progress. The visual progress bar can help users get clear information that how the donation progress went.

The 'Read More' link and 'See more projects' can bring the users to another page that have more information about the projects, such as: how the project will work, donation amount and times. I used a lighter blue that will not interapt users reading. But they can still see it when they finish reading the title and descriptions.

In the final prototype, I added an image with messages on it. It shows statistics that there are 60 projects currently working. It emphasized the number also invited users to participate in these projects. The image use was important to explore the natural beauty in the Great Barrier Reef. From user feedbacks, the images will make them more willing to protect the environment.

For the mobile version there were not many changes. The projects were layout in a column that users can scroll down.

Donate Page

Because the login section in the footer area. So I did not put another login section in donate page. I decided to keep the donation buttons simple and show the donation amounts. When user click the button, the background colour and text colour will change oppositely to highlight the donation amount.

The texts on the top image emphasised that users' support will make a huge difference to the reef environment. The text colour was white that is clear to see on the image. Also the text will not affect the viewing of images.

The texts in the bottom image also appreciated the donors. There are also some useful tips below that users can take everyday action to save the reef environment. This is also a call to action that let users know other than donate, their actions are important to help the reef.

In the mobile version, the donation amount buttons were in two columns. In the iteration process, I was thinking about make them in one column. However, it didn't work well that users need to scroll to very end to see the 'Donate NOW' button.
